💼 二、MPF计划配置（SPRO路径）

2.1 创建MPF供款计划

📋 配置步骤

路径: SPRO → Payroll → Payroll: China → Payroll China
     → MPF Configuration → MPF Plans → Create MPF Plan

操作步骤:
1. 点击"New Entries"创建新计划
2. 输入MPF计划代码（如: MPF_01）
3. 输入计划描述（如: Standard MPF Plan 5%）
4. 设置供款比例: 雇员5%, 雇主5%
5. 设置最低供款额: HKD 250
6. 设置最高供款额: HKD 1,500
7. 设置最低可课税收入: HKD 7,100
8. 设置最高可课税收入: HKD 30,000
9. 保存配置

2.2 MPF计划字段说明

字段 说明 示例值
MPF Plan Code MPF计划代码 MPF_01
Description 计划描述 Standard MPF 5%
Employee Rate 雇员供款比例 5.00
Employer Rate 雇主供款比例 5.00
Min Contribution 最低供款额 250
Max Contribution 最高供款额 1500
Min Relevant Income 最低可课税收入 7100
Max Relevant Income 最高可课税收入 30000

2.3 绑定MPF计划到人事子范围

🔗 绑定步骤

路径: SPRO → Payroll → Payroll: China → Payroll China
     → MPF Configuration → MPF Plans → Assign Plan to PSA

操作步骤:
1. 选择人事子范围（Personnel Subarea）
2. 点击"Assign"按钮
3. 从列表中选择MPF计划（如: MPF_01）
4. 设置生效日期（Valid From）
5. 保存配置

注意: 每个人事子范围只能绑定一个MPF计划

💰 三、工资类型配置

3.1 创建MPF相关工资类型

📋 必需的工资类型

工资类型代码 描述 类型
/MPF MPF可课税收入汇总 技术性
/MPF_EE 雇员MPF供款 扣款
/MPF_ER 雇主MPF供款 雇主成本
MPEX MPF豁免额 技术性

3.2 配置工资类型属性

🛠️ 配置路径与步骤

路径: SPRO → Payroll → Payroll: China → Payroll China
     → Wage Types → Create Wage Type

操作步骤:
1. 输入工资类型代码（如: /MPF）
2. 输入描述（如: MPF Relevant Income）
3. 设置工资类型分类（Wage Type Class）
   - 技术性类型: Technical
   - 扣款类型: Deduction
   - 雇主成本: Employer Cost
4. 设置计算类型（Calculation Type）
   - 汇总: Add
   - 扣减: Subtract
   - 比例: Percentage
5. 设置允许输入权限（Authorization）
6. 设置工资类型组（Wage Type Group）
7. 保存配置

3.3 配置工资类型估值规则

🔢 估值规则配置

路径: SPRO → Payroll → Payroll: China → Payroll China
     → Wage Types → Valuation Rules

示例: /MPF_EE (雇员MPF供款)
  - 规则类型: PCR (Personnel Calculation Rule)
  - PCR代码: XMPF (自定义MPF计算规则)
  - 计算方式: 供款基数 × 5%
  - 最低限额检查: 是
  - 最高限额检查: 是

示例: /MPF_ER (雇主MPF供款)
  - 规则类型: PCR
  - PCR代码: XMPF (共享计算规则)
  - 计算方式: 供款基数 × 5%
  - 最低限额检查: 是
  - 最高限额检查: 是

⚙️ 四、PCR和Schema配置

4.1 创建MPF计算PCR规则

📋 PCR规则代码示例

PCR代码: XMPF (MPF Contribution Calculation)

* XMPF - Calculate MPF Contribution
*
* 输入: /MPF (可课税收入)
* 输出: /MPF_EE (雇员供款), /MPF_ER (雇主供款)

* 检查是否有可课税收入
AMT /MPF
? ZERO => SKIP (无收入则跳过)

* 检查收入是否达到最低门槛
AMT /MPF
< 7100 => SET MINS (设置最低供款)

* 计算供款基数（取值范围: 7,100 - 30,000）
AMT /MPF
< 7100 => MIN 7100
> 30000 => MAX 30000

* 计算雇员供款（5%）
* 乘法系数: 0.05
MULTI ANA 0.05
ADDWT /MPF_EE

* 计算雇主供款（5%）
MULTI ANA 0.05
ADDWT /MPF_ER

* 检查最低供款限制（250 HKD）
AMT /MPF_EE
< 250 => AMT=250

* 检查最高供款限制（1,500 HKD）
AMT /MPF_EE
> 1500 => AMT=1500

* 同步调整雇主供款
AMT /MPF_EE
AMT /MPF_ER

* 结束

4.2 PCR规则配置步骤

🛠️ SPRO配置路径

路径: SPRO → Payroll → Payroll: China → Payroll China
     → Personnel Calculation Rules → Create PCR

操作步骤:
1. 输入PCR代码（如: XMPF）
2. 输入PCR描述（如: MPF Contribution Calculation）
3. 选择PCR分类（如: Deduction Calculation）
4. 编写PCR规则代码（见上例）
5. 语法检查（Validate Syntax）
6. 激活规则（Activate）
7. 保存配置

注意: PCR规则编写需要遵循SAP ABAP语法

4.3 更新Schema流程

🔄 Schema配置步骤

路径: SPRO → Payroll → Payroll: China → Payroll China
     → Schemas → Edit Schema

操作步骤:
1. 选择薪酬Schema（如: HK00）
2. 定位到扣款计算部分
3. 插入PCR调用: PUN XMPF
4. 确保PCR在正确位置执行
   - 通常在工资类型汇总后
   - 在净工资计算前
5. 保存Schema
6. 激活Schema

示例Schema片段:
  * Payroll Schema - Hong Kong
  *
  INPU   * 输入工资类型
  ADDWT  * 汇总可课税收入
  *
  PUN XMPF * 调用MPF计算PCR
  *
  NET    * 计算净工资

🔧 六、常见配置错误及修复

6.1 错误1: 供款金额计算错误

❌ 症状

  • 供款金额不符合5%规则
  • 最低/最高限制未生效

✅ 可能原因

  • PCR规则中的系数错误（应为0.05）
  • 最低/最高检查逻辑缺失
  • 工资类型估值规则配置错误

🔧 解决方案

  • 检查PCR规则XMPF中的乘法系数
  • 验证最低/最高检查逻辑是否存在
  • 使用T-Code: PE04检查PCR规则
  • 重新测试薪酬核算

6.2 错误2: 工资类型未计算

❌ 症状

  • /MPF_EE和/MPF_ER金额为0
  • 净工资计算错误

✅ 可能原因

  • Schema中未调用PCR规则
  • PCR规则未激活
  • 工资类型未正确绑定

🔧 解决方案

  • 检查Schema中是否有PUN XMPF调用
  • 使用T-Code: PE01检查Schema配置
  • 确认PCR规则状态为Active
  • 验证工资类型在PCR中被正确引用

6.3 错误3: 最低门槛未生效

❌ 症状

  • 月薪低于7,100 HKD时仍计算供款
  • 供款金额低于250 HKD

✅ 可能原因

  • MPF计划配置中最低可课税收入未设置
  • PCR规则中缺少最低收入检查
  • 最低供款检查逻辑缺失

🔧 解决方案

  • 更新MPF计划配置：Min Relevant Income = 7,100
  • 在PCR规则中添加最低收入检查
  • 添加最低供款检查逻辑（250 HKD）
  • 重新测试边界值场景

6.4 错误4: 人事子范围未绑定MPF计划

❌ 症状

  • 特定员工的MPF供款为0
  • 错误日志提示MPF计划未找到

✅ 可能原因

  • 员工所属人事子范围未绑定MPF计划
  • MPF计划绑定日期晚于员工入职日期
  • 人事子范围配置错误

🔧 解决方案

  • 检查员工的人事子范围（T-Code: PA20）
  • 确认人事子范围已绑定MPF计划
  • 调整MPF计划生效日期
  • 如需要，更新员工人事子范围
